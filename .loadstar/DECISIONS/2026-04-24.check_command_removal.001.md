### SOURCE
- WP: W://root/maintenance/remove_check_cmd
- Question: `loadstar check` 명령을 계속 유지할 것인가?

### CONTEXT
`loadstar check`는 초기에 "코드와 WayPoint 간 정합성 검증"을 목표로 설계되었으나, 해당 의도는 이미 폐기되었다. 현재 이 명령의 실질 기능은:

1. 마지막 git 커밋 시각 조회
2. WP 파일 modTime 비교
3. OUTDATED 판정 (mtime 기준)
4. 최근 수정 WP 상위 10개 표시

즉 "아직 커밋되지 않은 WP가 있는가"를 알려주는 정도로 축소됨. mtime 기반 판정은 checkout/rebase 등에 의해 쉽게 덮어써져 **가짜 양성**이 빈번.

### OPTIONS
1. **완전 제거** — 코드·WP·CLAUDE.md·README에서 모두 제거
2. **UI 대시보드 전용으로 내부 API화** — CLI 노출은 제거하되 내부 로직은 UI가 사용
3. **`loadstar show --recent 10` 옵션으로 흡수** — 기능 중복이지만 명령 수 감소

### DECISION
**Option 1 (완전 제거).**

### RATIONALE
- 현재 기능은 `git status .loadstar/` + `loadstar show` 조합으로 완전 대체 가능
- "느슨한 관리(Tolerable Consistency)" 방침과 맞지 않는 강제 검증 도구
- mtime 기반 판정은 신뢰도가 낮아 오히려 노이즈 생성
- 프로토타입은 명령 수를 최소화해 학습 허들 낮추는 편이 유리

### IMPACT
- 코드 제거:
  - `cmd/check.go` 삭제
  - `cmd/root.go`의 `AddCommand(checkCmd)` 제거
- 메타 제거:
  - `W://root/cli/cmd_check` WP 삭제
  - `root.cli` MAP WAYPOINTS 갱신
- 문서 정리:
  - `loadstar_cli/CLAUDE.md`: 세션 시작 절차 3번(check 실행) 제거 + 명령어 목록 갱신
  - `loadstar_ui/CLAUDE.md`: 세션 시작 절차 3번 제거
  - `loadstar_SPEC/README.md`: 명령어 목록에서 `check` 제거
  - `08.CLI_SPEC` 삭제된 명령어 이력에 추가

### COMMENT
향후 "코드↔WP 정합성 검증"이 정말 필요하다면, 구조적 접근(예: WP CODE_MAP scope의 파일 존재 여부 검사)으로 재설계할 것. mtime 기반은 재도입하지 않는다.
</DECISION>
